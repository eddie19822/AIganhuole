# 易剪 · 安装包打包与放到服务器分发

## 1. 本地打包出安装文件

在项目根目录执行：

```bash
npm install
npm run build
```

这会依次完成：TypeScript 检查 → Vite 打包前端与 Electron 主进程 → **electron-builder** 打安装包。

打包完成后，产物在 **`release/<版本号>/`** 目录（当前版本号见 `package.json` 的 `version`），例如 `release/0.1.0/`。

### 常见文件名（便于上传服务器）

| 平台 | 给用户安装用的文件 |
|------|-------------------|
| Windows | `易剪_<版本>_x64.exe`（NSIS 安装向导）；或 **`易剪_<版本>_x64.zip`**（解压后运行 `易剪.exe`，无需安装） |
| macOS | `易剪_<版本>_mac_universal.dmg`（Intel / Apple 芯片通用）、可选 `.zip` |
| Linux | `易剪_<版本>_x64.AppImage`（需在有图形环境的 Linux 上打包） |

### 只打某一平台（已执行过依赖安装）

```bash
npm run build:renderer && npm run pack -- --mac      # 仅 macOS
npm run build:renderer && npm run pack -- --win     # 仅 Windows
npm run build:renderer && npm run pack -- --linux   # 仅 Linux
```

也可用脚本：`npm run release:mac` / `release:win` / `release:linux`。

若在 **Mac 上打 Windows 包**时，`.exe` 安装向导失败（多为下载 NSIS 工具链访问 GitHub 超时），可只打 **zip 分发包**：

```bash
npm run release:win-zip
```

产物为 `release/<版本>/易剪_<版本>_x64.zip`，用户解压后双击运行即可。

### 打包注意事项

- **首次打包**会从网络下载对应平台的 Electron 二进制，请保证网络畅通。
- **macOS 通用包（universal）**需在 macOS 上执行 `release:mac`，且耗时更长。
- **Windows `.exe` 安装包**：在 **Windows 本机**或网络稳定环境下打包最稳妥；Mac 交叉编译依赖 Wine + NSIS，易因网络失败；zip 包不依赖 NSIS，可先在 Mac 上产出。
- **代码签名 / 公证**：上架 Mac App Store 或降低 Windows SmartScreen 告警需要各自平台的开发者证书；当前配置为**未签名**，用户首次打开可能需在系统设置里选择「仍要打开」。

## 2. 上传到服务器给用户下载

1. 把 **`release/<版本号>/`** 里需要的安装文件（如 `.exe`、`.dmg`、`.AppImage`）上传到服务器静态目录或对象存储。
2. 在网页上提供**直链下载**（建议使用 HTTPS）。
3. 可选：为同一版本附带说明文本（版本号、最低系统要求：Windows 10 64 位、macOS 10.15+ 等）。

用户无需 Node.js，只需下载对应系统的安装包并运行安装程序即可。

## 3.（可选）自动更新

项目内已接入 `electron-updater`。若以后要自动更新，需在打包配置里配置 `publish`（如 generic 服务器地址），并在服务端按 electron-builder 要求提供 `latest.yml` 等元数据。手动分发安装包时**可以不配置**，不影响离线安装使用。

## 4. 应用标识

- **AppId**：`com.yijian.desktop`（`electron-builder.json`）
- **显示名称**：`易剪`（`package.json` 的 `productName`）

更换标识时请同步修改上述两处及图标等资源。
